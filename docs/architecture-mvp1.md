# Wiggle Web MVP 1 데이터·권한·복구 모델

## 불변 결정

- `wiggle_web`은 `wiggle_draw`와 독립된 저장소다. Flutter 코드는 수정·복사하지 않는다.
- 수업 코드와 학급 QR은 입장 수단일 뿐 작품 접근 권한이 아니다.
- 모든 작품은 서버가 발급한 익명 `student_id` 소유이며 D1에 기록한다.
- PNG 썸네일과 최종 이미지는 R2에 두고 D1에는 소유자, 키, 버전 메타데이터만 둔다.
- 브라우저 저장소에는 기기 세션 토큰, 공유 태블릿 프로필 목록, 전송 전 오프라인 작업 큐만 둔다.
- AI 키와 AI 호출은 브라우저에 없다. MVP 1은 `CoachingEvent.actor=ai` 확장점만 예약한다.

## 권한 경계

| 주체 | 인증 | 허용 범위 |
|---|---|---|
| 교사 | 운영 SIWC 사용자 헤더, localhost 전용 HttpOnly 개발 세션 | 자신이 담당한 학급, 학생 상태·썸네일, 메시지 |
| 학생 | `Authorization: Bearer <device token>` | 자신의 프로필·작품·소감, 소속 학급 메시지 |
| 입장 전 사용자 | 활성 수업 코드 또는 QR join token | 새 익명 프로필 생성만 |
| 가족 | MVP 1 비대상 | 접근 불가 |

모든 원본 토큰은 한 번만 브라우저에 전달한다. D1에는 SHA-256 토큰 해시만 저장한다. 교사 PIN과 그림 비밀번호는 `nodejs_compat`의 비동기 `node:crypto`를 사용한 개인 salt PBKDF2-SHA256(100,000회) 결과만 저장해 Sites 운영 런타임 상한을 준수한다.

## 입장과 복구

1. 교사가 학급을 만들면 코드와 고엔트로피 join token이 발급된다.
2. 입장이 열려 있을 때 학생이 코드/QR로 진입해 별명·동물·그림 비밀번호 세 개를 정한다. 같은 그림을 반복해도 된다.
3. 서버가 학생 ID, 2시간 활성 기기 토큰, 개인 복구 QR token을 만든다.
4. 같은 브라우저 탭은 sessionStorage의 짧은 활성 세션으로 이어지고, 공유 태블릿 프로필을 다시 고르면 그림 비밀번호를 확인한다.
5. 다른 기기에서는 학급 코드+별명+동물+그림 비밀번호 또는 개인 QR token으로 새 기기 토큰을 발급받는다. 새 비밀번호는 세 개가 기본이고, 기존 네 개 비밀번호도 명시적인 이전 방식 선택으로 계속 복구한다.
6. 공유 태블릿은 여러 기기 토큰을 프로필 선택 화면에 표시하고, 선택한 프로필만 활성 요청에 사용한다.
7. 로그아웃은 활성 프로필만 해제한다. 수업 종료 시 학생 선택 화면으로 돌아간다.

## 저장 및 동기화

- 캔버스 동작은 좌표를 0~1로 정규화한 JSON으로 D1에 저장한다.
- 저장할 때 낮은 해상도 PNG를 R2에 올리고, 완료 시 최종 PNG와 불변 버전을 만든다.
- 네트워크 실패 시 요청 payload를 기기별 오프라인 큐에 넣고 `online` 이벤트에서 순서대로 재전송한다.
- 서버는 작품 소유권과 단조 증가하는 버전 번호를 검사한다. 점선 가이드는 별도 캔버스 레이어이며 작품 동작·PNG에 합성하지 않는다.

## 개인정보 최소화

- 학생 이메일·실명·학교명은 수집하지 않는다.
- 표시 별명과 동물만 학급에 보인다.
- 채팅 원문을 일반 로그로 쌓지 않는다. 메시지·소감·작품 버전처럼 제품에 필요한 구조화 사건만 보존한다.
- 점수, 순위, 재능 판정 필드는 스키마에 두지 않는다.

## 운영 전 보완

데모 교사 계정은 로컬 검증용이다. 실제 학교 배포 전에는 검증된 교사 IdP, 교사 초대/회수, 보존 기간·삭제 정책, 보호자 동의, 감사 로그, rate limiting, CSRF 방어와 키 회전 정책을 별도 보안 검토한다.
