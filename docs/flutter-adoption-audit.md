# Flutter 참고 자료 이식 감사

`wiggle_draw`는 읽기 전용 참고 자료다. 이 문서는 코드를 복사하기 위한 목록이 아니라, 검증된 제품 규칙을 웹에서 다시 구현하기 위한 경계다.

## 개념으로 이식한 규칙

- 작품 문서는 고정 `1024 × 1024` 좌표계를 사용하고 화면 크기와 분리한다.
- 순서형 op 로그는 `schemaVersion`, `rendererVersion`, `opId`, `clientOpId`, ISO timestamp, pressure를 가진다.
- 현재 MVP 렌더러는 Stroke를 제공하고 스키마는 Fill/Shape/Sticker 확장점을 명시한다.
- 문서 좌표에서 이전 점과 2.5 미만인 이동점은 추가하지 않는다.
- 256 PNG 썸네일과 1024 PNG 최종 이미지를 별도로 R2에 저장한다.
- 점선·교사 가이드는 작품 op와 분리된 캔버스 레이어이며 결과 PNG에 합성하지 않는다.
- 펜 굵기는 8/16/30, 팔레트는 기존 검증 색상 12개를 사용한다.
- 그리미는 자동 개입하지 않고 한 번에 질문 하나, 짧은 말, 실제 다음 행동, `그냥 그릴래` 탈출을 제공한다.
- 따라그리기는 6~15단계, 열린 선택 2개 이상, 마지막 자유 창작으로 구성한다.
- 스티커 식별자는 서버 `STICKER_ALLOWLIST` 밖 값을 거절한다.

## 웹에서 새로 구현한 부분

- D1 기반 익명 학생·학급·작품·복구·교사 메시지 모델
- R2 기반 썸네일·최종 이미지 저장
- HttpOnly 교사 세션과 학생 기기 Bearer 세션, 공유 태블릿 프로필 전환
- IndexedDB 오프라인 저장 큐, 멱등 request ID와 revision 충돌 검사
- 브라우저 Pointer Events 기반 터치·스타일러스 캔버스
- 낮은 빈도의 교사용 썸네일 진행실

## 폐기하거나 이식하지 않은 부분

- Flutter 위젯, 내비게이션, 로컬 파일 저장 구현
- 클라이언트 OpenAI 호출과 API 키 처리
- timestamp 기반 객체 ID
- 출생연도·학생 이메일·실명 수집
- 점수, 순위, 재능 판정, 자동 그림 완성
- 허가가 확인되지 않은 참고 이미지와 배경 이미지

기존 `draw_ai` 선택지 정규화에서 `다 아니야`가 중복될 수 있는 결함은 이식하지 않았다. 신규 코칭 확장점에서는 선택지 문자열을 정규화한 뒤 중복 제거하고, 특별 탈출 선택지는 정확히 한 번만 허용하도록 서버 스키마 테스트에서 고정한다.

승인된 파일과 해시는 [asset manifest](../public/brand/asset-manifest.json)에 기록한다.
